# プロダクト仕様

## 目的

自宅や研究室が所有する本を登録し、検索、編集、保管場所の管理を行えるWebアプリケーションを作成します。

## 対象ユーザー

- 自宅の本を整理したい個人
- 研究室の蔵書を管理したい小規模チーム
- 本のタイトル、著者、ISBN、バーコード、保管場所を後から検索したい人
- 管理用バーコードを使って蔵書を棚卸し、移動、確認したい人
- セルフホスト環境で蔵書データを自分たちで管理したい人

## 解決したい課題

- 所有している本を忘れて同じ本を重複購入してしまう
- 本の保管場所を簡単に確認できない
- 本に付いているバーコードと、独自に貼る管理用バーコードを別々に扱えない
- バーコードから書誌情報を取得できず、登録作業が手間になる
- 自宅や研究室で運用するためのデータ移行、バックアップ手段がない
- 紙やスプレッドシート管理では検索や更新が面倒

## MVPスコープ

最初のリリースでは、自宅や研究室で使う単一インスタンスの蔵書CRUDアプリとして完成させます。MVPではログイン機能を持たず、セルフホスト環境内での利用を前提にします。

### 含める機能

- 本の登録
- 本の一覧表示
- 本の詳細表示
- 本の編集
- 本の削除
- タイトル、著者、ISBN、本のバーコード、管理用バーコードによるキーワード検索
- 本に付いているバーコードの登録
- 管理用バーコードの登録
- カメラによるバーコード読み取り連携
- 本のバーコードから外部APIで書誌情報を取得
- 保管場所タグの事前登録
- 保管場所タグによる置き場所管理、絞り込み
- 分類タグによる分類、絞り込み
- Open Library APIのsubject情報を分類タグ候補として利用
- 蔵書データのExport
- 蔵書データのImport
- 管理メモの登録

### 含めない機能

- ユーザー登録、ログイン
- 複数ユーザー管理
- ネイティブAndroidフロントエンド
- 表紙画像アップロード
- 貸出管理
- 感想、レビュー、読書履歴の管理
- 欲しい本リスト
- 蔵書分類以外の汎用的なタグ管理
- レビュー公開やSNS連携

## 成功条件

- 手元の本をWeb画面から登録できる
- 登録済みの本を一覧で確認できる
- タイトル、著者、ISBN、本のバーコード、管理用バーコードで目的の本を探せる
- 本のバーコードから書誌情報を取得して登録作業を減らせる
- 管理用バーコードで蔵書を識別できる
- 保管場所タグで本の置き場所を管理できる
- 分類タグで本の種類を管理できる
- Export/Importでセルフホスト環境間の移行やバックアップができる
- 本の情報を編集、削除できる

## 初期技術方針

- フロントエンドとバックエンドを分離します。
- 初期フロントエンドはWebアプリケーションとして実装します。
- 将来的にフロントエンド部分をAndroidアプリケーションへ置き換え、または追加できるよう、バックエンドAPIは特定のフロントエンド実装に依存しない設計にします。
- データベースは初期開発ではローカル開発しやすいSQLiteを第一候補とします。
- APIはREST形式を第一候補とします。
- 書誌情報取得用の外部APIとしてOpen Library APIを利用します。
- 外部APIは入力補助として扱い、APIキー必須の仕組みには依存しません。
- 自宅や研究室でのセルフホスト運用を前提にします。
- HTTP環境のWebブラウザではカメラ利用に制約があるため、将来的にAndroidフロントエンドを追加しても同じバックエンドAPIを利用できる設計にします。
- セルフホストはDocker Composeを標準構成とします。
- SQLiteデータベースファイルは標準で `/data/book-manager.sqlite` に配置し、Docker volumeまたはホストディレクトリへマウントして、そのファイルをバックアップ対象とします。
- HTTPS対応は利用者のセルフホスト環境やリバースプロキシ方針に委ねます。
- 認証はMVPでは実装せず、将来拡張として扱います。

## 用語

- 本: 管理対象となる書籍、漫画、雑誌、技術書など
- 本のバーコード: 書籍自体に印刷されているISBN/JANなどのバーコード
- 管理用バーコード: 利用者が独自に本へ貼付する、蔵書管理用の一意なバーコード
- バーコード入力元: Web画面、キーボード入力型バーコードリーダー、将来のAndroidフロントエンドなど、バーコード値をアプリへ渡す手段
- 保管場所タグ: 事前登録した本の置き場所。例: リビング本棚、仕事部屋、倉庫
- 分類タグ: 本の内容や分野を表すタグ。Open Library APIのsubject情報、または手入力で登録する
- 管理メモ: 感想ではなく、破損、付属品、棚卸し補足など管理上のメモ
