# プロダクト仕様

## 目的

個人が所有する本を登録し、検索、編集、読書状態の管理を行えるWebアプリケーションを作成します。

## 対象ユーザー

- 自宅や職場の本を整理したい個人
- 読んだ本、未読の本、読書中の本を把握したい人
- 本のタイトル、著者、ISBN、メモを後から検索したい人

## 解決したい課題

- 所有している本を忘れて同じ本を重複購入してしまう
- 未読、読書中、読了の状態を簡単に確認できない
- 本に関するメモや保管場所が散らばっている
- 紙やスプレッドシート管理では検索や更新が面倒

## MVPスコープ

最初のリリースでは、個人利用を前提にした単一ユーザー向けの蔵書CRUDアプリとして完成させます。

### 含める機能

- 本の登録
- 本の一覧表示
- 本の詳細表示
- 本の編集
- 本の削除
- タイトル、著者、ISBNによるキーワード検索
- 読書ステータス管理
- 所有ステータス管理
- 本へのメモ登録

### 含めない機能

- ユーザー登録、ログイン
- 複数ユーザー管理
- ISBN外部API連携
- バーコード読み取り
- 表紙画像アップロード
- 貸出管理
- 高度なタグ管理
- レビュー公開やSNS連携

## 成功条件

- 手元の本をWeb画面から登録できる
- 登録済みの本を一覧で確認できる
- タイトル、著者、ISBNで目的の本を探せる
- 読書状態と所有状態を更新できる
- 本の情報を編集、削除できる

## 初期技術方針

- フロントエンドとバックエンドを分離します。
- データベースは初期開発ではローカル開発しやすいSQLiteを第一候補とします。
- APIはREST形式を第一候補とします。
- 認証はMVPでは実装せず、将来拡張として扱います。

## 用語

- 本: 管理対象となる書籍、漫画、雑誌、技術書など
- 読書ステータス: 未読、読書中、読了、積読、再読したい
- 所有ステータス: 所有中、欲しい、手放した
