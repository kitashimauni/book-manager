# プロダクト仕様

## 目的

個人または小規模なチームが所有する本を登録し、検索、編集、保管場所の管理を行えるWebアプリケーションを作成します。

## 対象ユーザー

- 自宅や職場の本を整理したい個人
- 本のタイトル、著者、ISBN、バーコード、保管場所を後から検索したい人
- 管理用バーコードを使って蔵書を棚卸し、移動、確認したい人

## 解決したい課題

- 所有している本を忘れて同じ本を重複購入してしまう
- 本の保管場所を簡単に確認できない
- 本に付いているバーコードと、独自に貼る管理用バーコードを別々に扱えない
- バーコードから書誌情報を取得できず、登録作業が手間になる
- 紙やスプレッドシート管理では検索や更新が面倒

## MVPスコープ

最初のリリースでは、個人利用を前提にした単一ユーザー向けの蔵書CRUDアプリとして完成させます。

### 含める機能

- 本の登録
- 本の一覧表示
- 本の詳細表示
- 本の編集
- 本の削除
- タイトル、著者、ISBN、本のバーコード、管理用バーコードによるキーワード検索
- 本に付いているバーコードの登録
- 管理用バーコードの登録
- 本のバーコードから外部APIで書誌情報を取得
- 保管場所タグの事前登録
- 保管場所タグによる分類、絞り込み
- 管理メモの登録

### 含めない機能

- ユーザー登録、ログイン
- 複数ユーザー管理
- 表紙画像アップロード
- 貸出管理
- 感想、レビュー、読書履歴の管理
- 欲しい本リスト
- 本以外の汎用的なタグ管理
- レビュー公開やSNS連携

## 成功条件

- 手元の本をWeb画面から登録できる
- 登録済みの本を一覧で確認できる
- タイトル、著者、ISBN、本のバーコード、管理用バーコードで目的の本を探せる
- 本のバーコードから書誌情報を取得して登録作業を減らせる
- 管理用バーコードで蔵書を識別できる
- 保管場所タグで本の置き場所を管理できる
- 本の情報を編集、削除できる

## 初期技術方針

- フロントエンドとバックエンドを分離します。
- データベースは初期開発ではローカル開発しやすいSQLiteを第一候補とします。
- APIはREST形式を第一候補とします。
- 書誌情報取得用の外部APIを利用します。
- 認証はMVPでは実装せず、将来拡張として扱います。

## 用語

- 本: 管理対象となる書籍、漫画、雑誌、技術書など
- 本のバーコード: 書籍自体に印刷されているISBN/JANなどのバーコード
- 管理用バーコード: 利用者が独自に本へ貼付する、蔵書管理用の一意なバーコード
- 保管場所タグ: 事前登録した本の置き場所。例: リビング本棚、仕事部屋、倉庫
- 管理メモ: 感想ではなく、破損、付属品、棚卸し補足など管理上のメモ
