# 仕様決定事項

このドキュメントは、実装前に決めておくべき仕様上の論点を管理します。

## 決定済みの仕様方針

### 1. 蔵書管理に集中する

このアプリは、読書記録や欲しい本リストではなく、所有している本を管理するための蔵書管理アプリとして設計します。

決定済み:

- 感想、レビュー、読書履歴はMVPに含めない
- 欲しい本リストはMVPに含めない
- 読了日、読書ステータスはMVPに含めない
- 管理メモは感想ではなく、破損、付属品、棚卸し補足など管理用途に限定する

### 2. 本のバーコードと管理用バーコードを分ける

本に印刷されているバーコードと、利用者が独自に貼付する管理用バーコードを別項目として扱います。

決定済み:

- 本のバーコードは書誌情報取得と検索に使う
- 管理用バーコードは蔵書の一意識別と検索に使う
- 管理用バーコードは外部API照会には使わない
- 管理用バーコードは入力された場合、一意である必要がある

### 3. 外部APIで書誌情報を取得する

本のバーコードを読み込んだとき、外部APIから書誌情報を取得できるようにします。

決定済み:

- MVPに外部API連携を含める
- 取得できた書誌情報は登録フォームに自動入力する
- 外部APIで見つからない場合も手入力登録できる
- 外部APIの失敗で本登録全体をブロックしない

未決定:

- 利用する外部APIプロバイダー
- 外部APIの取得結果をどの程度正規化するか
- 複数候補が返った場合の選択UI

### 4. 保管場所は事前登録タグとして扱う

保管場所は自由入力ではなく、事前に登録した保管場所タグから選択します。

決定済み:

- MVPに保管場所タグ管理を含める
- 本の登録、編集では保管場所タグを選択する
- 既に本に紐づく保管場所タグは削除せず、無効化する
- MVPでは1冊の本に設定できる保管場所タグは1つとする

## 実装前に残っている仕様論点

### 1. 本の分類単位

同じタイトルの本を、版、巻、媒体ごとに別の本として扱うかを決めます。

推奨案:

- ISBNが異なる本は別の本として扱う
- 漫画やシリーズ本は巻ごとに別の本として登録する
- 紙の本と電子書籍は必要に応じて別の本として登録する

決定状況:

- 未決定

### 2. 著者の複数入力

複数著者をどのように入力、表示、検索するかを決めます。

推奨案:

- MVPでは1つの文字列として保持する
- 入力時はカンマ区切りを許可する
- 検索は著者文字列への部分一致で行う

決定状況:

- 暫定決定済み

### 3. バーコード読み取り方法

Webアプリでどの読み取り方法をMVP対象にするかを決めます。

推奨案:

- MVPではバーコードリーダーがキーボード入力として値を送る方式を前提にする
- カメラ読み取りはPhase 2以降に回す

決定状況:

- 暫定決定済み

### 4. 外部APIプロバイダー

書誌情報取得に使う外部APIを決めます。

推奨案:

- 日本語書籍を優先するため、第一候補はopenBDとする
- openBDで取得できない場合に備えて、補助候補としてGoogle Books APIを検討する
- 楽天ブックス系APIはアプリIDが必要なため、MVPでは必須にしない
- API停止や取得失敗に備えて、手入力登録を必ず残す
- プロバイダー固有のレスポンスはバックエンド側でアプリ内形式へ変換する

候補:

- openBD: `https://openbd.jp/`
- Google Books API: `https://developers.google.com/books/docs/v1/using`
- Rakuten Books Book Search API: `https://webservice.rakuten.co.jp/documentation/books-book-search`

決定状況:

- 未決定

### 5. 保管場所タグの初期データ

初期状態で保管場所タグを用意するかを決めます。

推奨案:

- 初期データはなし
- ユーザーが最初に保管場所タグを登録する
- 開発、テスト用のサンプルデータは本番データとは分ける

決定状況:

- 未決定

### 6. 空状態の振る舞い

本や保管場所タグが未登録のとき、または検索結果が0件のときの表示を決めます。

推奨案:

- 本が未登録のときは本登録画面への導線を表示する
- 保管場所タグが未登録のときは保管場所登録画面への導線を表示する
- 検索結果0件時は検索条件の解除導線を表示する

決定状況:

- 未決定

## 先に決めなくてよい仕様

以下はMVP実装中に必要になってから決めます。

- 表紙画像の扱い
- カメラによるバーコード読み取り
- CSVインポート、エクスポート
- 貸出管理
- 複数ユーザー、認証、権限
- 棚卸し専用ワークフロー

## 実装前の最小決定セット

仕様面では、最低限次の3点が決まれば実装に入れます。

1. 本の分類単位
2. 外部APIプロバイダー
3. 保管場所タグの初期データ有無
