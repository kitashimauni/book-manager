# 仕様決定事項

このドキュメントは、実装前に決めておくべき仕様上の論点を管理します。

## 実装前に決めるべき仕様

### 1. 本の分類単位

同じタイトルの本を、版、巻、媒体ごとに別の本として扱うかを決めます。

推奨案:

- ISBNが異なる本は別の本として扱う
- 漫画やシリーズ本は巻ごとに別の本として登録する
- 紙の本と電子書籍は必要に応じて別の本として登録する

決定状況:

- 未決定

### 2. 著者の複数入力

複数著者をどのように入力、表示、検索するかを決めます。

推奨案:

- MVPでは1つの文字列として保持する
- 入力時はカンマ区切りを許可する
- 検索は著者文字列への部分一致で行う

決定状況:

- 暫定決定済み

### 3. 読書ステータスの初期値

本を登録した直後の読書ステータスを決めます。

推奨案:

- 初期値は `unread`
- 購入済みで読む予定の本はユーザーが `stacked` に変更する

決定状況:

- 暫定決定済み

### 4. 所有ステータスの初期値

本を登録した直後の所有ステータスを決めます。

推奨案:

- 初期値は `owned`
- 欲しい本リストとして登録する場合のみ `wishlist` に変更する

決定状況:

- 暫定決定済み

### 5. 欲しい本の扱い

まだ所有していない本をMVPで登録対象に含めるかを決めます。

推奨案:

- `ownership_status = wishlist` として登録可能にする
- 欲しい本も一覧、検索、詳細表示の対象に含める
- 本の登録フォームは所有中の本と共通にする

決定状況:

- 未決定

### 6. 読了日の扱い

読了日をMVPで持つかを決めます。

推奨案:

- MVPでは読了日は持たない
- 読書履歴や読了日の管理はPhase 2以降に回す

決定状況:

- 未決定

### 7. タグとジャンル

本をジャンルやタグで分類するかを決めます。

推奨案:

- MVPではタグ、ジャンルは実装しない
- 必要な補足情報はメモに書く
- 検索や整理で不足を感じたらPhase 2でタグ管理を追加する

決定状況:

- 未決定

### 8. 保管場所の粒度

保管場所を自由入力にするか、選択式にするかを決めます。

推奨案:

- MVPでは自由入力にする
- 例: `リビング本棚`, `仕事部屋`, `電子書籍`, `実家`
- 保管場所マスタは作らない

決定状況:

- 未決定

### 9. 削除時の安全性

削除確認の強さを決めます。

推奨案:

- MVPでは確認ダイアログのみ
- 誤削除防止を強める必要が出たら論理削除や復元機能を追加する

決定状況:

- 暫定決定済み

### 10. 空状態の振る舞い

本が1冊も登録されていないとき、または検索結果が0件のときの表示を決めます。

推奨案:

- 未登録時は登録画面への導線を表示する
- 検索結果0件時は検索条件の解除導線を表示する

決定状況:

- 未決定

## 先に決めなくてよい仕様

以下はMVP実装中に必要になってから決めます。

- 表紙画像の扱い
- ISBN外部API連携
- CSVインポート、エクスポート
- 読書履歴
- 貸出管理
- 複数ユーザー、認証、権限

## 実装前の最小決定セット

仕様面では、最低限次の4点が決まれば実装に入れます。

1. 欲しい本をMVPに含めるか
2. 読了日をMVPに含めるか
3. タグ、ジャンルをMVPに含めるか
4. 保管場所を自由入力にするか
