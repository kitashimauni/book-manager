# 機能仕様

## 画面一覧

| 画面 | パス案 | 説明 |
| --- | --- | --- |
| 本一覧 | `/books` | 登録済みの本を一覧表示し、検索と絞り込みを行う |
| 本登録 | `/books/new` | 新しい本を登録する |
| 本詳細 | `/books/:id` | 1冊の詳細情報を表示する |
| 本編集 | `/books/:id/edit` | 登録済みの本を編集する |
| 保管場所管理 | `/locations` | 保管場所タグを登録、編集、無効化する |
| 分類タグ管理 | `/classification-tags` | 分類タグを登録、編集、無効化する |
| データ入出力 | `/data` | 蔵書データのExport/Importを行う |

MVPではトップページ `/` は `/books` に誘導します。

## 本登録

### 入力項目

| 項目 | 必須 | 説明 |
| --- | --- | --- |
| タイトル | 必須 | 本の名前 |
| 著者 | 任意 | 複数名はカンマ区切りで入力 |
| 出版社 | 任意 | 出版社名 |
| 出版日 | 任意 | `YYYY-MM-DD` 形式 |
| ISBN | 任意 | ISBN-10またはISBN-13。書誌情報取得に使う正規化済みの値 |
| 本のバーコード | 任意 | 実際に読み取った本のバーコード値。ISBN由来の書籍JANコード、雑誌コード、価格コードなどを含み得る |
| 管理用バーコード | 任意 | 利用者が独自に貼付する管理用バーコード |
| 保管場所 | 任意 | 事前登録した保管場所タグから選択 |
| 分類タグ | 任意 | 外部APIのsubject、または事前登録した分類タグから選択 |
| 管理メモ | 任意 | 破損、付属品、棚卸し補足などの管理用メモ |

### バーコードによる書誌情報取得

- ISBNまたはISBNとして解釈できる本のバーコードを読み込むと、外部APIから書誌情報を取得します。
- 外部APIはNDLサーチAPIを優先し、見つからない場合のみOpen Library APIへフォールバックします。
- 本のバーコードがISBNとして解釈できる場合は、NDLサーチOpenSearch APIで完全一致照会します。
- 本のバーコードがISBNとして解釈できない場合は、読み取り値として保存し、書誌情報は手入力します。
- NDLサーチで取得できない場合は、Open Library ISBN APIで完全一致照会します。
- 完全一致で取得できた場合は、その書誌情報をフォームに自動入力します。
- NDLサーチ、Open Libraryのどちらでも取得できない場合は、候補検索へフォールバックせず手入力登録に進みます。
- MVPでは外部APIによる候補選択UIは実装しません。
- 取得対象はタイトル、著者、出版社、出版日、ISBN、subjectを基本とします。
- 取得できた項目は登録フォームに自動入力します。
- 取得できたsubjectは分類タグ候補として表示します。
- 分類タグ候補は自動保存せず、ユーザーが選択したものだけ外部API由来タグとして保存して本に紐づけます。
- 取得できなかった場合も、手入力で登録できます。
- 外部APIは入力補助であり、外部APIが利用できない状態でも手入力で本を登録できます。
- 管理用バーコードは書誌情報取得には使わず、蔵書の一意識別に使います。
- ISBN、読み取りバーコード、管理用バーコードの違いは、入力欄付近のヘルプで説明します。
- MVPのバーコード入力は、バーコードリーダーがキーボード入力として値を送る方式を前提にします。
- カメラによるバーコード読み取りは優先事項として扱います。

### ISBN、JAN、バーコードのヘルプ

本登録、編集画面では、ISBNと本のバーコード入力欄の近くにヘルプボタンを配置します。

ヘルプでは以下を説明します。

- 本のバーコードには複数種類があります。
- 日本の書籍の2段バーコードでは、上段がISBN由来の書籍JANコード、下段が価格などを含むコードであることが多いです。
- 書誌情報を取得したい場合は、ISBN欄へ入力するか、ISBNとして解釈できる上段バーコードを読み取ります。
- 雑誌や古い本など、ISBNではないバーコードもあります。その場合は本のバーコードとして保存し、書誌情報は手入力します。
- 管理用バーコードは利用者が独自に貼る蔵書管理用IDであり、書誌情報取得には使いません。

ISBNではない値を読み取った場合は、本のバーコード欄に値を反映し、「ISBNではない可能性があります。書誌情報取得にはISBNまたは上段バーコードを読み取ってください。」という案内を表示します。

### カメラ読み取り連携

- 初期フロントエンドはWebアプリケーションとして提供します。
- HTTPSなどブラウザのカメラ利用条件を満たす環境では、Web画面内でのカメラ読み取りを優先します。
- HTTP環境ではブラウザのカメラ利用に制約があるため、Web画面内のカメラ読み取りだけに依存しません。
- Web画面のカメラ読み取りは、対応ブラウザの `getUserMedia` と `BarcodeDetector` APIを利用します。
- 読み取り結果は本のバーコードまたは管理用バーコードのどちらへ入力するかをユーザーが選択できます。
- カメラ読み取りに対応しないブラウザや権限がない環境では、手入力またはキーボード入力型バーコードリーダーを使えるよう案内します。
- MVPでは、読み取り結果を本のバーコードまたは管理用バーコードの文字列として受け取れるAPI設計を優先します。
- 将来的にAndroidフロントエンドを追加する場合も、Webフロントエンド専用の内部状態に依存せず、同じバックエンドAPIへ接続できるようにします。
- バーコード読み取り、書誌情報取得、本登録は分離し、フロントエンド実装がWebでもAndroidでも既存の本登録APIを使い回せるようにします。
- AndroidフロントエンドはMVP対象外ですが、HTTP環境でカメラ利用が必要になった場合の有力な選択肢として想定します。

### バリデーション

- タイトルは1文字以上、200文字以下
- ISBNは未入力、ISBN-10、ISBN-13のいずれかを許可
- 本のバーコードは未入力、または任意の読み取り値を許可
- 管理用バーコードは未入力を許可するが、入力された場合は一意である必要があります
- 出版日は未入力、または有効な日付のみ許可
- 分類タグは登録済みのタグのみ紐づけできます
- 管理メモは5000文字以下

## 本編集

登録時と同じ項目を編集できます。

### 保存時の振る舞い

- 入力値が有効な場合は保存し、詳細画面へ戻ります。
- 入力値が不正な場合は保存せず、エラーメッセージを表示します。

## 本削除

- 削除前に確認ダイアログを表示します。
- MVPでは物理削除とします。
- 将来的に削除履歴が必要になった場合は論理削除へ変更します。

## 保管場所管理

### 表示項目

- 保管場所名
- 説明
- 表示順
- 有効、無効

### 操作

- 保管場所タグを登録する
- 保管場所タグを編集する
- 使わなくなった保管場所タグを無効化する
- 無効化した保管場所タグを再有効化する

### 振る舞い

- 本の登録、編集では有効な保管場所タグのみ選択できます。
- 既に本に紐づいている保管場所タグは削除せず、無効化します。
- 誤って無効化した場合や再利用する場合は、有効状態へ戻せます。
- MVPでは1冊の本に設定できる保管場所タグは1つとします。

## 分類タグ管理

### 表示項目

- 分類タグ名
- 説明
- 有効、無効

### 操作

- 分類タグを登録する
- 分類タグを編集する
- 使わなくなった分類タグを無効化する
- 無効化した分類タグを再有効化する

### 振る舞い

- NDLサーチまたはOpen Library APIから取得したsubjectは分類タグ候補として表示します。
- ユーザーが確定したsubjectのみ分類タグとして保存します。
- 本の登録、編集では複数の分類タグを紐づけできます。
- 既に本に紐づいている分類タグは削除せず、無効化します。
- 誤って無効化した場合や再利用する場合は、有効状態へ戻せます。

## データ入出力

### Export

- 本、保管場所タグ、分類タグをまとめてJSON形式でExportします。
- JSON Exportは、同じアプリのImportで復元できる正式なバックアップ、移行形式とします。
- 他ツールでの閲覧、分析向けにはCSV Exportを別機能として追加できるようにします。
- CSV Exportは復元用の正規形式ではなく、Excel、LibreOffice、Python、Rなどで確認、分析するための読み取り用形式として扱います。
- CSV ImportはMVPでは必須にせず、必要になった時点で列仕様と重複解決ルールを別途決めます。

### Import

- ExportしたJSONファイルをImportできます。
- Import前にバリデーション結果を表示します。
- Import前に追加、上書き候補、無視候補、エラーをプレビュー表示します。
- 既存データと重複する場合は、Windowsでフォルダをコピーしたときのように、個別に上書きするか無視するかを選択できます。
- 重複項目に対して `すべて上書き` と `すべて無視` を選択できます。
- 重複項目の初期選択は `無視` とし、意図しない上書きを避けます。
- 保管場所タグ名、分類タグ名、管理用バーコードなどの重複は、Import重複解決ルールに従います。
- MVPでは既存データを全削除する置換Importは行わず、追加Importと選択的な上書きを基本とします。

## バックアップ

- セルフホスト環境では、SQLiteデータベースファイル `/data/book-manager.sqlite` をDocker volumeまたはホストディレクトリへマウントします。
- 通常のバックアップは、マウントされたSQLiteデータベースファイルをバックアップする運用を基本とします。
- Export/Importは、アプリケーション上でのデータ移行や部分的な取り込みに使います。
